# Milestone 1 Plan (Canvas Core)

## 목표
`roadmap.md`의 **M1. Canvas Core** 5개 항목을 한 번에 충족한다.
- React Flow 기반 캔버스/노드/엣지 기본 동작
- 최대 `10x10` 기준 그리드 모델 및 점유 로직
- 가시 영역 단계 확장(`4x4 -> 7x7 -> 10x10`)
- 경계 밖 드래그 시 다음 단계 자동 확장 UX
- 드래그 시작/종료 시 그리드 가이드 표시/숨김

## 범위 (In Scope)
- React Flow 기본 화면, 노드/엣지 상태, 연결/삭제 상호작용
- 그리드 상태 모델(`gridSize`, `occupiedCells`, `visibleStage`) 도입
- 셀 점유/해제 규칙(노드 이동 시 재계산 포함)
- 단계 확장 트리거 및 뷰 갱신
- 경계 초과 드래그 감지 후 자동 확장
- 드래그 중 가이드 오버레이 표시, 종료 시 숨김

## 범위 (Out of Scope)
- 스냅 히스테리시스/`lastValidDock` 고도화(M2)
- 블록 타입별 상세 편집 UX(M4)
- 저장/복원 및 협업(M6)
- 외부 검색 API 연동(M5)

## 구현 단계

### 1. Canvas Baseline
- React Flow 캔버스 초기화, 노드/엣지 상태 훅 연결
- 노드 생성/이동, 엣지 연결/삭제 기본 이벤트 연결
- 확대/축소는 버튼 기반으로 제한하고 미니맵 제외

완료 기준
- 노드/엣지 기본 조작이 오류 없이 동작
- 기본 캔버스 interaction(선택, 패닝, 줌)이 안정적

### 2. Grid Model (`10x10` Max)
- 셀 좌표 체계와 점유 맵 타입 정의
- 노드 위치 -> 셀 매핑 유틸 구현
- 점유 충돌/경계 체크 순수 함수화

완료 기준
- 현재 노드 배치에서 점유 셀이 일관되게 계산
- 유효/무효 셀 판정이 테스트 가능한 함수로 분리

### 3. Stage Expansion (`4x4 -> 7x7 -> 10x10`)
- 초기 가시 영역을 `4x4`로 시작
- 확장 조건 충족 시 `7x7`, 이후 `10x10`으로 단방향 확장
- 단계별 렌더링 범위와 가이드 범위 동기화

완료 기준
- 단계가 역행 없이 순차 확장
- 단계 전환 후 기존 노드/엣지 상태 무결성 유지

### 4. Auto Expand on Boundary Drag
- 드래그 중 노드가 현재 가시 경계 밖으로 벗어나는지 감지
- 경계 초과 시 다음 단계 자동 확장 트리거
- 확장 직후 드래그 흐름이 끊기지 않게 이벤트 처리

완료 기준
- 경계 밖 드래그 시 즉시 다음 단계로 확장
- 확장 이후에도 노드 이동/연결 상호작용 정상 유지

### 5. Grid Guide Visibility
- `onNodeDragStart`에서 가이드 오버레이 표시
- `onNodeDragStop`/취소 시 가이드 숨김
- 가이드 스타일을 단계 크기와 일치하도록 렌더

완료 기준
- 드래그 중에만 가이드가 보이고 종료 시 항상 숨김
- 빠른 연속 드래그에서도 표시 상태가 꼬이지 않음

### 6. 안정화 및 검증
- 상태 책임 분리(UI 이벤트 vs 도메인 계산)
- 핵심 계산 유닛 테스트(점유/경계/단계 전환)
- 수동 시나리오 점검 및 빌드 검증

완료 기준
- `pnpm build` 및 타입체크 통과
- M1 체크리스트 전 항목 수동 검증 통과

## 산출물
- React Flow 기반 Canvas Core 화면 1개
- `10x10` 최대 그리드 + 점유 로직 + 단계 확장 동작
- 경계 드래그 자동 확장 및 가이드 표시/숨김 UX
- M2로 넘길 수 있는 그리드/도킹 확장 가능한 구조

## 리스크와 대응
- 리스크: 뷰(UI 좌표)와 셀(도메인 좌표) 불일치로 점유 오류 가능
- 대응: 좌표 변환 유틸 단일화, 경계/점유 판정은 순수 함수로 고정

- 리스크: 단계 확장 트리거가 과민/과소 동작할 가능성
- 대응: 경계 감지 기준값을 상수화하고 드래그 이벤트 테스트 케이스 확보

- 리스크: 자동 확장 시 렌더 재계산으로 드래그 체감 끊김 발생 가능
- 대응: 확장 상태 업데이트를 최소화하고 가이드 렌더 비용을 분리
