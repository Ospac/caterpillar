# Milestone 1 Plan

## 목표
`roadmap.md`의 첫 번째 Milestone인 **"React Flow 기반 캔버스/노드/엣지 기본 동작"**을 구현한다.

핵심은 다음 3가지를 최소 기능으로 안정화하는 것이다.
- 캔버스에서 노드 생성/이동 가능
- 노드 간 엣지 연결/삭제 가능
- 현재 그래프 상태(노드/엣지)를 화면 상태로 일관되게 유지

## 범위 (In Scope)
- `reactflow` 라이브러리 도입 및 기본 Canvas 구성
- 최소 1개 커스텀 노드 타입(예: `textNode`) 정의
- 노드 추가(초기 버튼 또는 더블클릭 기반) 기능
- 노드 드래그 및 위치 반영
- 핸들을 통한 엣지 생성(`onConnect`) 및 엣지 삭제
- 상태 관리(useNodesState/useEdgesState 또는 내부 store) 정리
- 기본 UX(선택, 패닝, 줌, 미니맵/컨트롤 중 1~2개)

## 범위 (Out of Scope)
- 그리드 점유/자석/도킹 로직 (다음 Milestone)
- 블록 타입별 상세 편집 UI
- 저장/복원(LocalStorage/Cloud)
- 외부 검색 API 연동

## 구현 단계

### 1. 기술 준비
- `reactflow` 설치
- `features/diagram` 하위에 React Flow 전용 UI/모델 구조 정리
- 스타일 초기 세팅(`ReactFlow` 기본 CSS 포함)

완료 기준
- 빈 캔버스가 렌더링되고 에러 없이 개발 서버에서 동작

### 2. 최소 그래프 렌더링
- 초기 노드/엣지 샘플 데이터 정의
- `ReactFlow`에 노드/엣지 주입
- `Background`, `Controls` 등 기본 보조 UI 연결

완료 기준
- 초기 렌더링 시 노드 2개 + 엣지 1개가 보임
- 줌/패닝이 정상 동작

### 3. 상호작용 구현
- `onNodesChange`, `onEdgesChange` 연결
- 노드 드래그 반영
- `onConnect`로 새 엣지 생성
- 엣지 삭제 핸들러 연결(키보드 Delete 또는 UI 액션)

완료 기준
- 노드 이동 후 위치가 즉시 상태에 반영
- 노드 핸들 연결로 새 엣지 생성 가능
- 선택된 엣지 삭제 가능

### 4. 노드 생성 흐름
- 캔버스 상단 버튼 기반 `Add Node` 우선 구현
- 생성 시 뷰포트 중심 근처에 새 노드 배치
- 기본 라벨/ID 생성 규칙 적용

완료 기준
- 버튼 클릭 시 노드가 중복 ID 없이 지속적으로 생성
- 생성 직후 드래그/연결 가능

### 5. 코드 품질 정리
- 상태/이벤트 핸들러 분리 (UI와 모델 책임 분리)
- 타입 정의(`NodeData`, `BlockNodeType`) 보강
- 최소 스모크 테스트 시나리오 문서화

완료 기준
- `pnpm build` 통과
- 핵심 상호작용 수동 점검 체크리스트 통과

## 산출물
- React Flow 기반 다이어그램 화면 1개
- 노드/엣지 기본 조작 가능한 MVP
- 다음 Milestone(그리드 모델)로 확장 가능한 상태 구조

## 검증 체크리스트
- [ ] 노드 추가가 정상 동작한다.
- [ ] 노드 드래그 시 위치가 유지된다.
- [ ] 노드 간 연결 생성이 가능하다.
- [ ] 엣지 삭제가 가능하다.
- [ ] 캔버스 줌/패닝이 깨지지 않는다.
- [ ] 빌드/타입체크가 통과한다.

## 예상 작업 순서(권장)
1. 의존성 추가 + 기본 캔버스 렌더링
2. 노드/엣지 상태 연결
3. 생성/삭제 상호작용 구현
4. 타입/구조 정리 및 빌드 확인
